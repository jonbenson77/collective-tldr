Set Up a Three-Level Claude Memory System That Works Across All AI Tools

Jack walks through a tiered memory architecture he spent over a year testing, covering three levels of memory: short-term using Claude project files and CLAUDE.md, mid-term using Obsidian as a structured knowledge base, and long-term using Pinecone vector search for semantic retrieval across everything. He compares Pinecone versus Obsidian for different use cases and gives the exact folder structure and setup steps. The system works across Claude, Cursor, ChatGPT, and NotebookLM.

  • Level 1 Short-Term:: CLAUDE.md and project context files inside Claude for session-level memory
  • Level 2 Mid-Term:: Obsidian vault organized by project and domain for persistent reference across weeks
  • Level 3 Long-Term:: Pinecone vector database for semantic search across all past context when you have hundreds of files
  • Pinecone vs Obsidian:: Obsidian is simpler and sufficient for most users; Pinecone adds value only when your knowledge base exceeds easy manual navigation

Three-Level Setup Walkthrough
  • Level 1 (Short-term): Create a CLAUDE.md file at the root of every project folder. Write in your business context, constraints, tone rules, and recurring preferences. Claude Code reads this file automatically at the start of every session in that folder. For Claude Cowork, use the Project Instructions field to store the same information.
  • Level 2 (Mid-term): Set up an Obsidian vault with folders per domain: clients, automations, copy rules, CRM notes, funnel specs. Link notes between projects. When you start a new Claude session, copy the relevant Obsidian note content into the conversation or reference the file path if using Claude Code with file access.
  • Level 3 (Long-term): Connect Pinecone to store embeddings of all your Obsidian notes and past project outputs. Use a Firecrawl or script-based ingestion to keep Pinecone updated. Query it via Claude Code when you need to pull relevant past context semantically rather than by exact file name.
Why Most Memory Systems Fail and How This One Avoids It

Jack opens at 0:00 by identifying the core failure mode: most people store context in one place but only access it from one tool, so context switching between Claude, Cursor, and ChatGPT means starting over every time. His fix is to treat Obsidian as the single source of truth that all tools read from, either by manual paste for simpler sessions or via Pinecone embeddings for automated retrieval. The second failure mode is over-engineering Level 3 before you need it. His recommendation is to start with just CLAUDE.md for one project this week, add Obsidian after you hit your first context loss, and only add Pinecone once your vault exceeds around 50 notes.
